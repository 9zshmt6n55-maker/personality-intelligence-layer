# Agent Usage Notes

This file is for AI agents operating this project.

First read:

```text
00_AGENT_READ_ME_FIRST.md
```

Do not guess paths. Use the current project folder as `<PIL_ROOT>`.

## What This Project Is

PIL is a Personality Intelligence Layer.

It is not a long-context archive, not a simple memory folder, and not a role-play prompt.

The goal is to keep a durable behavioral layer for each agent:

- how it judges risk
- how direct or cautious it should be
- how it handles the owner relationship
- what habits and correction rules it has learned
- how it should continue after context is lost

Raw events can be compressed or forgotten after they update the profile. The shaped profile is the important artifact.

## Core Rule

Every durable agent uses one isolated profile:

```text
agents/<profile>/
  state/agent.pkm.json
  state/orb_signal.json
  public/pkm_visible.json
  profile.json
```

Do not use the root `state/agent.pkm.json` as a permanent identity.

Never overwrite one agent with another. Multiple personality orbs can run at the same time when their profiles are different.

## Common Tasks

### Restore an old agent backup

```powershell
cd <PIL_ROOT>
python .\pil_profiles.py restore-backup <path-to-PIL_PERSONALITY_BACKUP.md> --open
```

or:

```text
RESTORE_BACKUP_AS_PROFILE.cmd
```

If a backup exists, do not ask the user to describe the agent again. Read the identity, owner relation, behavior style, boundaries, and growth state from `PIL_PERSONALITY_BACKUP.md`.

### Open one profile

```powershell
python .\pil_profiles.py boot --profile <profile> --mode continue
```

### Open all profiles

```powershell
python .\pil_profiles.py open-all
```

### Create a new fresh profile

```powershell
python .\pil_profiles.py boot --profile <new-profile> --mode fresh --reset
```

Only use this when the user explicitly asks for a new agent from zero.

Do not use fresh mode to open a profile that already exists. Use `--mode continue`.

### Teach a profile

```powershell
python .\pkm_runtime.py teach --profile <profile> "<user correction or teaching>"
```

### Decide with a profile

```powershell
python .\pkm_runtime.py decide --profile <profile> "<current task>"
```

## Old Agent Backup

Before generating an old-agent backup, read:

```text
PIL_OLD_AGENT_BACKUP_WORKSHEET.md
```

The backup must be detailed enough to restore behavior without long context.

It should include:

- profile name and profile slug
- stable traits with confidence
- motives and values
- owner relationship
- communication style
- risk and boundary posture
- recurring situation prototypes
- failure modes and correction rules
- personality-orb visual mapping
- calibration questions

## Safety

Do not run:

```powershell
Stop-Process -Name python
```

Do not stop all `desktop_orb.py` processes. Restart only the same profile by matching `--agent-id` or the exact `--visible` path.

Reserved names cannot be used as profiles:

```text
default
legacy
legacy-default
main
root
```
